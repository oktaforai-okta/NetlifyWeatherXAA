XAA Weather API

This API provides weather data protected by Okta XAA (eXternal Agent Access) token authentication.

Endpoint: POST /.netlify/functions/weather

Authentication

Include your XAA token in the Authorization header:

Authorization: Bearer <your-xaa-token>

Request Body

{
    "city": "San Francisco",
    "state": "CA"
}

Example Response

{
    "city": "San Francisco",
    "state": "CA",
    "temperature": 65.5,
    "feelsLike": 63.2,
    "humidity": 72,
    "condition": "Clouds",
    "conditionDescription": "scattered clouds",
    "windSpeed": 12.5,
    "weatherDescription": "Current weather in San Francisco, CA: 65.5°F...",
    "success": true
}

Error Responses

  • 401 - Invalid or missing XAA token
  • 400 - Missing city or state in request
  • 500 - Internal server error
